   output pass; drop scarlet-only syntax (`> 900px`, `animation` block, `const`).
3. **Sequencing:** prove an end-to-end vertical slice first, then expand.
4. **Module identity:** rebrand to **`github.com/rohanthewiz/go-styl`**, drop the
   `codetree` dependency and the hardcoded local `replace` directive.

## Architecture built

```
.styl
  â†’ parser (indentation line-tree + comment strip + Pratt expression parser)
  â†’ ast
  â†’ eval (lexical scope, variable inlining, arithmetic, builtins, control flow,
          functions/mixins via a context-passing execBlock)
  â†’ css (rule tree â†’ render; optional duplicate merge)

```

Package layout (all under `internal/` except the public API):

| Path | Role |
|---|---|
| `styl.go` | public API: `Compile`, `CompileReader`, `CompileFile`, `Options` |
| `cmd/styl/main.go` | CLI: `-o`, `-compress`, `-merge` |
| `internal/token` | token kinds + `Token` |
| `internal/lexer` | expression/value tokenizer (numbers+units, colors, strings, ops, `...`) |
| `internal/ast` | statement + expression nodes |
| `internal/parser` | line-tree builder, comment stripping, statement classifier, Pratt expr parser |
| `internal/value` | `Number/Color/Str/Ident/Bool/Null/List` + unit-aware arithmetic |
| `internal/eval` | scope chain + `execCtx`/`execBlock` evaluator + selector flattening |
| `internal/builtin` | function registry (`rgb`, `rgba`, `unit` so far) |
| `internal/css` | output rule tree, renderer, optional duplicate-merge |

### Key implementation refinement
Rather than a single token stream with INDENT/DEDENT feeding a full grammar, used
a **line-tree + expression-lexer hybrid**: selectors stay as robust raw strings
(avoids the `a[href$="a,b"]` tokenization trap), and only *values/conditions* are
tokenized and Pratt-parsed. Matches how Stylus's indentation mode behaves.

The eval layer uses a **context-passing `execBlock`** (`execCtx{scope, rule,
parents, ret, returned}`) so the same statement executor emits uniformly at top
level, inside a ruleset, and inside a function body. This is what lets a mixin
drop declarations into the caller's rule *and* spawn nested rulesets against the
caller's selector context.
