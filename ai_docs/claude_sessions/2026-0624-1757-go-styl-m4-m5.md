```
  → parser (brace→indent normalize, indentation line-tree + Pratt expr parser)
  → ast

```

## M4 — interpolation, `@extend`/`$placeholder`, `@import` (commit `580dbb8`)

### Interpolation `{expr}`
- **Lexer** (`lexer.go`): `scanIdent` folds balanced `{...}` groups into IDENT
  tokens, so `col-{i}`, `{prop}-top`, and bare `{x}` tokenize as one IDENT (braces
  kept). Added `{`/leading-`-{` dispatch + `skipInterp`.
- **Parser**: exported `parser.ParseExpr(src, line)` (lex+parse one expression),
  the bridge eval uses to resolve interpolation contents. (eval now imports
  parser — no cycle: parser imports only ast/lexer/token.)
- **Eval** (`internal/eval/m4.go`): `interpolate()` scans a raw string for balanced
  `{...}` and substitutes each group's evaluated CSS via `evalString`. Applied to:
  raw **selectors** (in `evalRuleSet`), **property names** (Declaration), **string
  literals** (StringLit), and **idents**. A lone `{expr}` ident yields the *value*;
  a mixed `weight-{n}` yields a substituted bare ident.

### `@extend` / `@extends` + `$placeholder`
- `ast.Extend{Target}`; parser handles `@extend`/`@extends` leaves.
- `css.Rule` gained `Selectors []string`, `Placeholder bool`, `Extenders []string`;
  `renderSelectors` joins own-selector (unless placeholder) + duplicates + extenders,
  returning "" for an unextended placeholder (renders nothing).
- Eval records `extendReq{extenders, target}` during the walk and resolves them in a
  **post-pass** (`applyExtends` → `findExtendTargets`): `$name` → placeholder
  registry; otherwise every rule whose `Selectors` contains the target. Forward
  references work because it's a post-pass. `$`-selectors register as placeholders.

### `@import`
- `ast.Import{Path, Literal}`; parser classifies `.css` / `url(...)` / absolute URLs
  as **literal passthrough**, everything else as an **inline** `.styl`.
- Eval (`evalImport`): literal → `css.RawNode` (`@import "x.css";`). Inline →
  `resolveImport` (searches `BaseDir` then `IncludePaths`, trying the path, `.styl`,
  and `index.styl`), reads+parses, then `execBlock`s the imported statements in the
  **current scope** (shares vars/mixins) with `dir` updated to the import's folder.
  Cycle guard via `ev.importing` set.
- `styl.Options` gained `BaseDir` (derived from `Filename` in `CompileFile`);
  `IncludePaths` now wired through.

### M4 tests
- `testdata/m4.{styl,css,min.css}` golden; `testdata/imports/{_vars,main}.styl`
  fixture (subdir not globbed by `TestGolden`).
- `m4_test.go`: 10-case feature table + `TestM4Import`. `lexer_test.go`:
  `TestLexInterpolation`.

## M5 — at-rules, brace syntax, compress parity (commit `42ed474`)

User scoped M5 to **at-rules + compress parity + brace syntax**, deferring
sourcemaps.

### Output model change (foundation)
- evaluator now builds an ordered `out []css.Node`; `execCtx` carries a
  `sink *[]css.Node` where rulesets/at-rules emit. `ev.rules []*css.Rule` still
  collects *every* rule (any nesting) for `@extend` lookup. Removed the separate
  `raws` bucket (literal imports now emit to the sink in encounter order).
- `invoke` refactored to take the caller's `*execCtx` as the emission context (so a
  mixin's nested rulesets land in the caller's sink); function calls get a scratch
  sink so emission never nil-derefs.
- `css.AtRule` changed `Rules []*Rule` → `Nodes []Node`; `Render` renders+indents
  its body and **skips empty** at-rules. `MergeDuplicates` now takes/returns
  `[]Node`, merging only top-level rules and passing at-rules through.

### At-rules (`internal/eval/m5.go`, parser `parseAtRule`)
- `ast.AtRule{Name, Params, Body}`; parser splits header into keyword (no `@`) +
  raw params; block form gets a parsed Body, leaf form Body=nil.
- Dispatch in `evalAtRule` (params run through `interpolate`):
  - **leaf** (no body) → `css.RawNode` `@name params;` (`@charset`, `@namespace`).
  - **`@font-face`/`@page`/`@viewport`** → declaration block: a `css.Rule` whose
    selector *is* the header (`@font-face { … }`).
  - **`@keyframes`** (+ vendor via `atKind` stripping `-x-`) → frames emitted as-is
    (parents reset, no `&` combine).
  - **default `@media`/`@supports`/…** → `evalAtNested`: nested rules in an
    `css.AtRule`; when `ctx.parents` is non-empty (nested inside a selector), it
    **bubbles** — a wrapper rule with the inherited selector receives bare decls.
- Compress: `compactAtHeader` drops spaces after `:`/`,` inside parens in
  compressed mode (`@media (min-width:768px)`).
- **Known limit:** bare variables in media params aren't evaluated — use
  interpolation `{bp}` (raw params otherwise).

### Brace / semicolon syntax (`internal/parser/braces.go`)
- `Parse` calls `usesBraces(src)` and, if so, `bracesToIndent(src)` to normalize
  CSS-like `{ } ;` source into the indentation line-tree — the rest of the parser is
  unchanged.
- A single `scanStructural` scanner walks runes outside strings/comments and emits
  events (text/interp/open/close/semi/newline). **Interpolation vs block** braces are
  disambiguated structurally: a `{...}` glued (no space) to an identifier-ish char on
  its left or right is interpolation; otherwise a block delimiter (`isInterpBrace`/
  `isGlue`). `usesBraces` only trips on a block whose body contains `\n`/`;`/`:`, so
  indentation files with value interpolation (`width {n}`) are **not** misdetected.
- **Known limit:** a stand-alone `{expr}` in value position isn't supported in brace
  mode (use the bare variable). Documented at the top of `braces.go`.

## Known carry-forward limitations

- Bare variables in `@media` params need `{…}` interpolation.
- Stand-alone `{expr}` value in brace mode unsupported (use bare variable).
- Unquoted `url(/path.ext)` still trips the value lexer (`.`/`/`) — quote it.
- Whitespace-sensitive `-`/`+` (always binary); `calc()` args get evaluated.
- Selector comma-split doesn't respect brackets/strings; no `!important`.
